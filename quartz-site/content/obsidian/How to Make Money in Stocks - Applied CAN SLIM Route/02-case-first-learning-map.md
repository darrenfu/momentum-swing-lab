# Case-First Learning Map

Source: [Alternative Source](https://github.com/pistolla/gnidart/blob/master/How%20to%20Make%20Money%20in%20Stocks%20-%20A%20Winning%20System%20in%20Good%20Times%20and%20Bad%204th%20edition%202009.pdf)

## 为什么要从案例开始

这本书最值钱的地方不是定义，而是历史样本。O'Neil 用跨越一百多年的大赢家图形说明：行业和产品会变，人性、供需、机构买入、突破和顶部行为会反复出现。

你的学习顺序应该是：

```text
先看故事和图 -> 标注价格与成交量 -> 抽出 CAN SLIM 概念 -> 写成自己的交易规则
```

## 每个案例只问 7 个问题

1. 这家公司当时的 `N` 是什么：新产品、新服务、新管理层、新高，还是新行业周期？
2. `C` 和 `A` 是否强：最近季度和多年盈利是否支持价格强势？
3. 它是否是 `L`：行业和市场中的领导股，还是落后股补涨？
4. `S` 是否成立：突破、回踩、加仓点是否有成交量证据？
5. `I` 是否健康：是否有机构需求，但还没有拥挤到明显派发？
6. `M` 是否配合：大盘处于进攻、震荡，还是应该防守？
7. 如果买错，哪一个价格或行为证明 thesis 失效？

## 从故事提炼概念的方法

| 故事情节 | 不要停留在 | 要提炼成 |
|---|---|---|
| 公司产品很新 | “这个故事很酷” | 新因素必须被收入、盈利、价格、成交量确认 |
| 股价创新高 | “太贵了”或“追高” | 新高可能是机构需求的证据，但必须来自正确 base |
| 突破时放量 | “今天涨很多” | 需求突然超过供给，买点有质量 |
| 回踩 10 周线 | “跌了所以危险” | 强势股第一次健康回踩可能是加仓或持有检验 |
| 大盘转弱 | “我的股票不一样” | M 是系统过滤器，不是背景噪音 |
| 小亏损出现 | “再等等” | 小亏损是交易成本，大亏损是系统损坏 |

## 案例学习节奏

- 每天 1-3 个案例，不追求数量。
- 每个案例必须画出 base、pivot、stop、sell signal。
- 每张卡只写 5-8 行，避免变成读书摘抄。
- 每周选 1 个案例，对照当下市场找一个相似候选股，只做观察，不急着交易。
