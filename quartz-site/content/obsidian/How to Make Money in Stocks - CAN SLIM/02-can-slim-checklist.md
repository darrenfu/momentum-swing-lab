# CAN SLIM Checklist

## 评分方式

每只候选股先按绿/黄/红判断，不要急着算复杂分数。

- 绿：证据明确，和系统一致。
- 黄：证据不完整，需要继续观察。
- 红：违反核心规则，直接剔除或等待重新形成。

## 候选股过滤

| 模块 | 绿灯 | 黄灯 | 红灯 |
|---|---|---|---|
| C - 当前季度 | 最近季度 EPS 和销售强增长，最好有加速 | 只 EPS 强或只销售强 | 增长停滞、一次性收益、销售不配合 |
| A - 年度盈利 | 过去数年盈利持续上升，ROE 有质量 | 年度波动但趋势改善 | 长期亏损或周期性假繁荣 |
| N - 新因素 | 新产品、新市场、新管理层、新高或新周期 | 有叙事但数据未确认 | 只有故事，没有业绩和价格确认 |
| S - 供需 | 突破时明显放量，流动性充足 | 量能尚可但不突出 | 低流动性、放量下跌、突破无量 |
| L - 领导股 | 相对强度领先，行业内排名靠前 | 跟随强势行业但不是第一梯队 | 弱势股补涨、低位便宜股 |
| I - 机构 | 有质量机构逐步参与，但未过度拥挤 | 机构信息不完整 | 无流动性或机构过度拥挤后开始撤退 |
| M - 市场 | 大盘处于确认上行或风险可控环境 | 市场震荡，轻仓试探 | 市场处于明显压力或下跌趋势 |

## 买入前检查

- [ ] 股票来自强行业或正在改善的行业。
- [ ] 不是因为“便宜”而买，而是因为增长、强势和供需证据。
- [ ] 图形存在合理 base，不是一路延伸后追高。
- [ ] pivot 或买点清晰，买入价没有离买点过远。
- [ ] 突破时成交量支持，或至少没有明显供给压力。
- [ ] 大盘环境允许做多。
- [ ] 已经写好止损价、仓位、加仓条件和卖出条件。
- [ ] 如果今天买入后立刻下跌，你知道哪一价位证明自己错了。

## 直接剔除

- 只因为低 PE、股息高、跌很多或看起来便宜而买。
- 向下摊平，试图用更低价格证明自己正确。
- 低价、低成交额、消息驱动但没有持续业绩和价格确认。
- 相对强度明显落后，却幻想它会补涨成龙头。
- 市场环境很差时重仓做多。
- 买入后没有写止损和复盘计划。

## 卖出和风控

### 亏损处理

O'Neil 的系统要求亏损要很快被切断。实践上可以把单笔硬止损放在买入价下方约 7%-8% 的区间，并根据你的买点质量、波动率和仓位进一步收紧。关键不是这个数字本身，而是不能让小错变成组合级灾难。

### 盈利处理

盈利仓位不要只靠感觉卖。至少提前定义：

- 什么情况下继续持有：突破后走势健康、回踩有支撑、市场环境仍配合。
- 什么情况下减仓：短期过度延伸、明显异常放量、接近预设目标或市场转弱。
- 什么情况下退出：破坏关键均线、跌回买点下方、出现连续异常派发、基本面证伪。

### 仓位公式

```text
单笔允许亏损金额 = 账户总权益 * 单笔风险比例
每股风险 = 买入价 - 止损价
股数 = 单笔允许亏损金额 / 每股风险
```

建议学习期先把单笔风险控制在很小的范围。目标不是快速赚钱，而是训练“规则一致性”。

## 加仓规则

- 只在第一笔盈利后加仓。
- 加仓价格应来自新的合理买点或健康回踩，不是因为下跌后想降低成本。
- 每次加仓都重新计算组合风险。
- 如果加仓后总仓位让你无法执行止损，说明仓位已经过大。

## 一页版 CAN SLIM

```text
强基本面 + 新因素 + 强相对强度 + 强行业 + 机构需求
        + 正确 base/pivot + 大盘支持 + 严格止损
```
